# 품질계약 (Quality Contract)

> AI가 생산하는 모든 산출물은 이 품질계약을 통과해야 납품 가능합니다.
> "팀장이 개입해야 한다면, 시스템이 실패한 것" — 매니페스토 원칙

---

## 1. 공통 품질 기준 (모든 산출물 적용)

| 기준 | 정의 | 검증 방법 | 불합격 사유 |
|------|------|----------|-----------|
| **정확성** | 데이터, 수치, 인용이 사실에 부합 | 출처 교차검증 | 수치 오류, 출처 불일치 |
| **완결성** | 요청된 모든 항목이 빠짐없이 포함 | 요구사항 체크리스트 대조 | 섹션 누락, 미완성 표 |
| **일관성** | 용어, 형식, 톤이 문서 전체에서 통일 | 용어집 대조, 형식 점검 | 약어 불일치, 톤 불통일 |
| **가독성** | 비전문가(팀장)도 즉시 이해 가능 | 가독성 자가 점검 | 전문 용어 미설명, 복잡한 문장 |
| **근거성** | 모든 주장에 데이터·이론·사례 근거 제시 | 근거 유무 확인 | "~해야 한다" 근거 없는 주장 |

## 2. 유형별 품질 기준

### 2-1. 프레젠테이션 (pptx)

| 기준 | 상세 | 합격 기준 |
|------|------|----------|
| **스토리라인** | 전체 흐름이 논리적으로 연결 | 슬라이드 순서대로 읽으면 하나의 이야기 |
| **1슬라이드 1메시지** | 각 슬라이드에 핵심 메시지 1개 | 제목 = 그 슬라이드의 결론 |
| **시각 비율** | 텍스트 vs 시각요소 비율 | 시각요소(차트/표/도식) ≥ 40% |
| **폰트 통일** | 제목/본문/주석 폰트 체계 | 3가지 이하 폰트 사용 |
| **색상 체계** | 색상 팔레트 일관 적용 | 3-5색 이내, CI 반영 |
| **레이아웃** | 정렬, 여백, 간격 통일 | 눈에 거슬리는 정렬 불일치 없음 |

### 2-2. 보고서 (docx)

| 기준 | 상세 | 합격 기준 |
|------|------|----------|
| **프레임워크** | 검증된 분석 프레임워크 활용 | SWOT, BSC, NCS 등 1개 이상 |
| **표/그림 번호** | 모든 표와 그림에 번호, 제목, 출처 | [표 1-1] 형식, 출처 명시 |
| **목차 일치** | 목차와 본문 페이지가 정확히 일치 | 자동 목차 기능 또는 수동 검증 |
| **참고문헌** | 본문에서 인용된 모든 출처 목록 | APA 또는 한국학술 형식 |

| **머리글/바닥글** | 프로젝트명 + 페이지 번호 | 본문 시작부터 번호 매김 |

| 기준 | 상세 | 합격 기준 |
|------|------|----------|

| **시트 구성** | 원시→정제→분석→요약 시트 구조 | 최소 3개 시트 이상 |
| **헤더 고정** | 첫 행 헤더 고정(틀 고정) | 스크롤 시 헤더 보임 |

| **데이터 유효성** | 비어있는 셀, 이상치 처리 | 결측치 표시, 이상치 주석 |
| **단위 명시** | 모든 수치 열에 단위 표기 | %, 원, 명, 건 등 |

## 3. 랄프 루프 (Ralph Loop) 연계

```
초안 생성 → Momus 1차 검증 → 수정 → Momus 2차 검증 → 확정
```

### 검증 프로세스
1. **1차 검증**: 공통 품질 기준 5항목 + 유형별 기준 전수 체크
2. **1차 수정**: 검증 피드백 반영하여 수정
3. **2차 검증**: 1차 지적사항 해소 여부 + 전체 재확인
4. **확정**: 2차 검증 통과 시 납품 가능

### 실패 처리
- **2회 수정 이내 통과**: 정상 — 산출물 납품
- **3회 이상 수정 필요**: 기획 단계 문제 — brief 보완 후 재시작
- **반복 실패 패턴**: Pygmalion 루프가 감지하여 스킬/프로세스 개선

## 4. 품질 등급

| 등급 | 기준 | 의미 |
|------|------|------|
| **A (우수)** | 공통 5항목 + 유형별 전항목 충족, 수정 0-1회 | 즉시 납품 가능 |
| **B (양호)** | 공통 5항목 충족, 유형별 1-2항목 미흡, 수정 1회 | 경미한 수정 후 납품 |
| **C (보통)** | 공통 3항목 이상 충족, 유형별 다수 미흡, 수정 2회 | 상당한 수정 필요 |
| **D (미흡)** | 공통 기준 미달 | 재작성 또는 기획 재검토 |

## 5. 사용 방법

이 품질계약은 다음과 같이 활용됩니다:

1. **산출물 생성 시**: 해당 유형의 품질 기준을 체크리스트로 사용
2. **Momus 검증 시**: 이 문서를 검증 기준으로 참조
3. **최종 납품 전**: 공통 + 유형별 기준 전수 확인
4. **품질 개선 시**: 반복 실패 항목을 식별하여 스킬 개선

## 주의사항
- 이 품질계약은 **최소 기준**이며, 프로젝트별로 추가 기준을 정할 수 있음
- 발주처가 별도 품질 요건을 제시한 경우, 그 기준을 우선 적용
- 품질 등급 D는 절대 납품 불가 — 반드시 재작업
- 품질계약 자체도 Pygmalion 루프의 개선 대상
