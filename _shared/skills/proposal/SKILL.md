# 제안서 작성 스킬

## 용도
컨설팅 사업 **제안서(Proposal)** 및 제안발표자료를 작성할 때 사용합니다. 공공기관 입찰, 민간기업 수주, 파트너십 제안 등 모든 유형의 제안서에 적용됩니다.

## 적용 조건
- "제안서 만들어줘", "제안서 작성", "입찰 제안서" 등의 요청
- RFP(제안요청서) 기반 제안서 작성 시
- 제안발표(PT) 자료 작성 시

---

## 제안서 유형 및 구조

### 유형 1: 공공기관 입찰 제안서 (pptx, 주력)
정부/공공기관 용역사업 입찰 시 제출하는 제안서. **RFP(제안요청서)의 평가기준을 반드시 반영**.

| 섹션 | 슬라이드 | 핵심 내용 |
|------|---------|----------|
| 표지 | 1p | 사업명, 제안사, 제출일 |
| 제안 개요 | 2-3p | 핵심 강점 요약, 차별화 포인트 3가지 |
| 사업 이해 | 3-5p | 배경, 필요성, 관련 정책/법률, 목적, 범위 |
| 추진 전략 | 5-8p | 방법론 프레임워크, 단계별 계획, 간트차트 |
| 품질 관리 | 2-3p | 품질관리 체계, 위험관리, 산출물 검수 |
| 수행 역량 | 3-5p | 기관 소개, 유사 실적, 투입 인력, 협력체계 |
| 기대 효과 | 1-2p | 정량적/정성적 효과, 활용 방안 |

### 유형 2: 민간 수주 제안서 (pptx 또는 docx)
기업 고객에게 컨설팅 서비스를 제안하는 문서.

| 섹션 | 분량 | 핵심 내용 |
|------|------|----------|
| Executive Summary | 1p | 고객 이슈 + 해결 방안 + 기대 효과 한 페이지 요약 |
| 고객 현황 분석 | 2-3p | 고객사 이슈 진단, 산업 트렌드, 경쟁 환경 |
| 제안 솔루션 | 3-5p | 구체적 접근방법, 방법론, 기대 산출물 |
| 수행 계획 | 2-3p | 일정, 투입 인력, 커뮤니케이션 계획 |
| 비용 제안 | 1-2p | 견적 구조, 옵션, 결제 조건 |
| 레퍼런스 | 1-2p | 유사 프로젝트 실적, 고객 추천사 |

## 섹션별 작성 가이드

### 표지
- 사업명은 RFP 원문 그대로 (임의 변경 금지)
- 제안사 CI/로고 포함
- "제안서" 또는 "기술제안서" 명시
- 제출 일자, 수신처(발주기관) 명시

### 제안 개요 (Executive Summary)
- **1-2 문장 핵심 메시지**: "본 제안의 핵심은 ~입니다"
- **3가지 차별화 포인트**: 왜 우리를 선택해야 하는가
  - 방법론 차별성 (어떻게 다르게 할 것인가)
  - 실적 차별성 (유사 경험이 있는가)
  - 인력 차별성 (전문가가 누구인가)
- **한 슬라이드에 핵심을 전달** — 이것만 읽어도 제안 전체를 이해할 수 있어야 함

### 사업 이해
- **발주처의 언어를 사용**: RFP에 나온 용어와 표현을 그대로 활용
- **법률/정책 근거**: 관련 법령, 정부 정책, 산업 동향 인용
- **문제 정의**: 현재 상황의 문제점을 구체적으로 제시
- **Tip**: 사업 이해 섹션에서 "우리가 이 사업을 깊이 이해하고 있다"는 인상을 줄 것

### 추진 전략 (핵심 섹션)
- **방법론 프레임워크**: 반드시 시각적 다이어그램으로 표현
  - 단계(Phase) → 활동(Activity) → 산출물(Output) 구조
  - 프로세스 흐름이 한눈에 보이도록 설계
- **단계별 상세 계획**: 각 단계의 목표, 방법, 도구, 산출물
- **간트차트**: 전체 일정 가시화 (주요 마일스톤 표시)
- **차별화 포인트 강조**: 타 제안사 대비 우위 요소를 명시적으로 표현

### 수행 역량
- **기관 소개**: 설립 연도, 전문 분야, 주요 고객 (1-2 슬라이드)
- **유사 실적**: 최근 3년 유사 프로젝트 (발주처, 사업명, 기간, 규모)
  - **핵심**: 평가기준에서 유사 실적 배점이 높으면 이 섹션 강화
- **투입 인력**: 학력, 경력, 자격증, 유사 프로젝트 참여 이력
  - PM/책임연구원은 상세 프로필, 연구원은 요약표
- **협력체계**: 공동수행/하도급 기관이 있는 경우 역할 분담 다이어그램

### 비용 제안 (민간 제안서)
- **견적 구조**: 인건비, 경비, 부가세 분리 표기
- **옵션 제공**: 기본형/표준형/프리미엄 3단계 견적 (선택의 여지 제공)
- **지불 조건**: 계약금 / 중도금 / 잔금 비율 (일반적으로 30-30-40)
- **가격 경쟁력**: 가격 자체보다 가성비(Value for Money) 강조

## 디자인 원칙

### 슬라이드 설계
- **슬라이드당 핵심 메시지 1개** — 제목 = 그 슬라이드의 결론
- **텍스트 최소화**: 키워드 + 도식/표 중심 (글씨 빽빽한 슬라이드 금지)
- **6×6 규칙**: 한 슬라이드에 6줄 이내, 한 줄에 6단어 이내 (발표용)

### 색상
- **발주처 CI 활용**: 가능하면 발주처(고객) 색상을 메인으로
- **대안**: 네이비/딥블루 계열 (가장 무난하고 전문적)
- **포인트 색상**: 1개만 (주황, 빨강 등으로 강조점만)
- **배경**: 흰색 또는 매우 연한 회색 (어두운 배경 지양)

### 폰트
- **제목**: 맑은 고딕 Bold, 24-28pt
- **본문**: 맑은 고딕 Regular, 14-18pt
- **주석/출처**: 맑은 고딕 Light, 10-12pt
- 영문이 섞일 경우: Arial 또는 Calibri

## RFP 분석 절차

제안서 작성 전 반드시 수행:

1. **RFP 숙독**: 사업 개요, 과업 범위, 제출 요건, 평가 기준 파악
2. **평가기준 매핑**: 기술평가 항목별 배점을 제안서 구성에 반영
   - 배점 높은 항목 → 슬라이드 수 + 내용 깊이 강화
   - 배점 낮은 항목 → 간결하게 필수 요건만 충족
3. **필수 요건 체크**: 제출 서류, 분량 제한, 형식 요건 확인
4. **경쟁사 분석**: 예상 경쟁사의 강점/약점 파악 → 차별화 전략 수립

## 품질 체크리스트

- [ ] RFP의 모든 평가 항목이 제안서에 반영되었는가
- [ ] 제출 양식/분량 제한을 준수했는가
- [ ] 방법론 다이어그램이 포함되었는가
- [ ] 간트차트(일정표)가 포함되었는가
- [ ] 유사 실적이 구체적으로 기술되었는가 (기관명, 기간, 규모)
- [ ] 투입 인력의 자격/경력이 명시되었는가
- [ ] 각 슬라이드에 핵심 메시지가 명확한가
- [ ] 오탈자 및 숫자 오류 검수가 완료되었는가
- [ ] 발주처 기관명이 정확한가 (약칭 사용 시 주의)
- [ ] 제안 가격이 정확하게 산출되었는가 (공공 입찰 시)

## 파일명 규칙
`[제안서]_[기관명]_[사업명]_v버전.pptx`

예시:
- `[제안서]_시앤피_선도기업직무디자인연구용역_v1.0.pptx`
- `[기술제안서]_시앤피_AX진단컨설팅도구개발_v2.1.pptx`
- `[제안서]_시앤피_직무분석훈련과정개발_최종.pptx`

## 산출물 형식
| 제안서 유형 | 권장 형식 | 비고 |
|------------|----------|------|
| 공공 입찰 기술제안서 | pptx | 발표 겸용 |
| 민간 제안서 | pptx 또는 docx | 고객 선호에 따라 |
| 비용 제안서 | xlsx + pptx | 산출근거 + 요약 |
| 사업계획서형 제안서 | docx | 투자/파트너십용 |

## 주의사항
- 발주처 기관명, 사업명은 반드시 공식 명칭 사용
- RFP에서 요구하지 않은 내용을 과도하게 넣지 말 것 (분량 낭비)
- 유사 실적은 검증 가능한 것만 기재 (허위 실적 기재 시 입찰 자격 박탈)
- 제안 발표 시간이 정해진 경우, 슬라이드 수 조절 (통상 1슬라이드=1-2분)
- pptx 작성 시 반드시 pptx 스킬 연계 사용
