# 컨설팅 보고서 작성 스킬

## 용도
직무분석, 산업전환(AX) 진단, 훈련과정 개발, 조직진단, HR 컨설팅 등 **컨설팅 프로젝트의 결과보고서**를 작성할 때 사용합니다. 한국 공공기관·민간기업 납품 기준에 맞는 전문적인 보고서를 생산합니다.

## 적용 조건
- "보고서 써줘", "결과보고서 작성", "컨설팅 보고서" 등의 요청
- 프로젝트 완료 후 납품용 최종 보고서 작성 시
- 중간보고서, 최종보고서, 착수보고서 등 모든 보고서 유형

|------|--------|----------|
| 표지 | 1p | 프로젝트명, 수행기관, 발주처, 일자 |
| 사업 개요 | 2-3p | 배경, 목적, 범위, 기대효과 |
| 추진 체계 | 2-3p | 조직도, 역할분담, 의사소통 체계 |
| 수행 방법론 | 3-5p | 프레임워크, 분석도구, 접근방법 |
| 추진 일정 | 1-2p | 간트차트, 마일스톤, 산출물 일정 |
| 기대 산출물 | 1p | 산출물 목록 및 형식 |

### 2. 중간보고서 (Interim Report)
| 섹션 | 페이지 | 핵심 내용 |
|------|--------|----------|
| 표지 | 1p | 프로젝트명, 수행기관, 보고일자 |
| 추진 현황 | 2-3p | 일정 대비 진행률, 주요 활동 |
| 주요 발견사항 | 5-10p | 분석 중간결과, 데이터 요약 |
| 이슈 및 조치사항 | 2-3p | 문제점, 위험요인, 대응방안 |
| 향후 계획 | 1-2p | 잔여 일정, 조정사항 |

### 3. 최종보고서 (Final Report) — 주력 산출물
| 섹션 | 페이지 | 핵심 내용 |
|------|--------|----------|
| 표지 | 1p | 프로젝트명, 수행기관, 발주처, 완료일자 |
| 목차 | 1-2p | 장·절·표·그림 목차 |
| 사업 개요 | 3-5p | 배경, 목적, 범위, 추진체계, 방법론 요약 |
| 수행 방법론 | 5-8p | 분석 프레임워크, 조사 방법, 도구, 일정 |
| 현황 분석 | 10-20p | 현재 상태 진단, 데이터 분석 결과 |
| 핵심 발견사항 | 10-15p | 주요 분석 결과, 시사점, 벤치마킹 |
| 개선 방안 | 10-15p | 과제 도출, 실행 로드맵, 우선순위 |
| 기대 효과 | 3-5p | 정량적/정성적 효과, KPI |
| 부록 | 별도 | 설문지, 인터뷰 가이드, 원시 데이터, 참고문헌 |

## 섹션별 작성 가이드

### 표지
- **필수 요소**: 프로젝트명, 수행기관 로고/명칭, 발주처명, 보고 일자, 보고서 유형(착수/중간/최종)
- **디자인**: 발주처 CI 색상 활용, 깔끔하고 공식적인 레이아웃
- **프로젝트명**: 공식 계약서상 사업명 그대로 사용 (임의 약칭 금지)

### 사업 개요
- **배경**: 왜 이 사업이 필요한지 (정책 동향, 법률 근거, 시장 변화)
- **목적**: SMART 기준으로 명확하게 (구체적, 측정가능, 달성가능, 관련성, 시한)
- **범위**: 무엇을 하고, 무엇을 하지 않는지 명확하게 구분
- **추진체계**: 조직도 + 역할분담표 (PM, 연구원, 자문위원 등)

### 수행 방법론
- **프레임워크 도식**: 반드시 시각적 다이어그램으로 표현
- **단계별 설명**: 각 단계의 목적, 방법, 산출물 명시
- **분석 도구**: 사용한 도구/소프트웨어 명시 (NCS DB, Excel, Python 등)
- **조사 방법**: 설문, 인터뷰, FGI, 문헌조사 등 방법별 대상·기간·규모

### 현황 분석
- **데이터 기반**: 모든 주장에 근거 데이터 제시
- **시각화 우선**: 표, 차트, 그래프 적극 활용 (본문 텍스트 최소화)
- **비교 분석**: 유사기관, 업계 평균, 과거 데이터와 비교
- **구조**: 분석 항목별로 "현황 → 분석 결과 → 시사점" 3단계 전개

### 핵심 발견사항
- **So What 원칙**: 단순 데이터 나열이 아닌, "그래서 무엇을 의미하는가" 해석
- **우선순위**: 발견사항을 중요도/시급도 매트릭스로 정리
- **인용 근거**: 인터뷰 발언, 설문 수치, 벤치마킹 데이터 등 출처 명시

### 개선 방안 (제안/권고사항)
- **과제 체계**: 전략과제 → 세부과제 → 실행과제 위계 구조
- **실행 로드맵**: 단기(3개월), 중기(6개월), 장기(1년+) 구분
- **우선순위**: 실행 용이성 vs 기대효과 매트릭스
- **구체성**: "~해야 한다"가 아닌, "~을 ~까지 ~방법으로 실행" 수준

### 기대 효과
- **정량적**: 비용절감 OO%, 생산성 OO% 향상, 이직률 OO% 감소 등
- **정성적**: 조직문화 개선, 직원 만족도 향상, 역량 체계화 등
- **KPI**: 측정 가능한 성과지표 3-5개 제시

## 시각화 기준

### 필수 시각화 요소
| 데이터 유형 | 권장 차트 | 비고 |
|------------|----------|------|
| 현황/비율 | 원형차트, 도넛차트 | 항목 5개 이하 |
| 비교 분석 | 가로/세로 막대 | 항목 비교 시 |
| 추이/변화 | 꺾은선 그래프 | 시계열 데이터 |
| 분포 | 히스토그램, 박스플롯 | 점수 분포 등 |
| 프로세스 | 플로우차트, 화살표 | 방법론, 절차 |
| 체계/구조 | 트리맵, 조직도 | 분류체계, 직무체계 |
| 일정 | 간트차트 | 추진일정 |
| 우선순위 | 2x2 매트릭스 | 중요도×시급도 |

### 시각화 규칙
- 모든 차트에 **제목, 범례, 단위, 출처** 명시
- 색상은 3-5색 이내, 색맹 고려 (빨-초 조합 지양)
- 텍스트 위주 슬라이드라도 **핵심 수치는 시각적 강조** (큰 숫자, 색상 박스)
- 표는 가독성 우선: 교차 줄 색상, 헤더 구분, 적절한 여백

## 톤 & 스타일

### 문체 규칙
- **공식적이고 전문적인 어조** — 경어체 사용, "~입니다/~됩니다" 종결
- 약어 처음 사용 시 풀네임 병기: NCS(국가직무능력표준), AX(인공지능전환)
- **능동태 우선**: "분석되었다" → "분석하였다", "조사된" → "조사한"
- **한문장 한메시지**: 한 문장에 하나의 핵심만 담을 것
- 불필요한 수식어 제거: "매우 중요한" → "핵심적인"

### 한국 보고서 관행
- **경칭 표기**: 발주처는 "귀 기관/귀 사" 또는 정식 명칭 사용
- **페이지 매김**: 좌측 하단 또는 중앙 하단, 본문 시작부터 넘버링
- **머리글**: 프로젝트명 + 수행기관명 (선택)
- **바닥글**: 페이지 번호 + "Confidential" 또는 보안등급 (필요 시)
- **표/그림 번호**: "[표 1-1]", "[그림 2-3]" 형식 (장번호-일련번호)
- **부록 표기**: 별도 페이지 번호 체계 또는 "부록-1, 부록-2" 형식
- **인쇄 고려**: A4 기준 여백 상하 25mm, 좌우 30mm (제본 여백 포함)

## 품질 체크리스트

### 최종 제출 전 확인 사항
- [ ] 프로젝트명이 계약서/제안서와 일치하는가
- [ ] 목차와 본문 페이지가 일치하는가
- [ ] 모든 표/그림에 번호, 제목, 출처가 있는가
- [ ] 수치 데이터의 합계가 100%인가 (비율 데이터)
- [ ] 오탈자 및 띄어쓰기 검수가 완료되었는가
- [ ] 약어가 처음 사용 시 풀어 쓰여 있는가
- [ ] 참고문헌이 본문에서 인용된 순서대로 정렬되었는가
- [ ] 부록이 본문에서 참조되었는가 (미참조 부록 불가)
- [ ] 기대효과에 구체적 수치가 포함되었는가
- [ ] 제안사항이 실행 가능한 수준으로 구체적인가

## 주의사항
- 모든 데이터와 인용에 출처를 반드시 명시
- 저작권 있는 자료(논문, 보고서)는 출처 표기 후 활용
- 발주처 내부 데이터는 "기업 제공 자료" 등으로 표기 (구체적 수치 공개 주의)
- 보고서 완성 후 반드시 맞춤법 검사 실행
- pptx 작성 시 pptx 스킬, docx 작성 시 docx 스킬 연계 사용
