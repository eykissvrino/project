## 3. 기존 코드에서 발견한 이슈 & 보완사항

### 🔴 즉시 수정 필요

#### 이슈 1: scheduler.ts가 notify.ts에 강하게 의존
```
현재: scheduler.ts → import { sendEmailNotification, sendSubscriberNotifications, sendDigestNotifications } from "./notify"
문제: 알림 제거 시 scheduler.ts가 깨짐
해결: scheduler.ts에서 알림 관련 import/호출 전부 제거, runKeywordNotify()를 키워드-결과 매칭 전용으로 리팩토링
```

#### 이슈 2: instrumentation.ts에서 스케줄러 시작
```
현재: register() → startScheduler() → node-cron 기반 digest 알림 포함
문제: node-cron 패키지 제거 시 instrumentation.ts 깨짐
해결: CRON을 Railway 외부 스케줄러(또는 /api/cron GET)로 대체, instrumentation.ts 단순화
```

#### 이슈 3: Sidebar에 "알림 설정" 메뉴 하드코딩
```
현재: navItems에 { href: "/settings", icon: Bell, label: "알림 설정" } 존재
해결: 제거 + 관리자 메뉴 추가 (role 기반 조건부 렌더링)
```

### 🟡 개선 권장

#### 이슈 4: 사전규격 4종 전부 크롤링 중 → 용역만 필요
```
현재: PRESPEC_ENDPOINTS 4개 (물품/용역/공사/외자)
     PRESPEC_SEARCH_ENDPOINTS 4개
해결: 용역(Servc) 1개만 남기고 3개 제거 → API 호출 75% 감소, 속도 대폭 향상
```

#### 이슈 5: 관리자 계정 구분 로직 미흡
```
현재: middleware.ts에서 role 체크 없음 (토큰 유효성만 검증)
     seed.ts에 role="admin"은 있지만 실제 접근 제어 없음
해결: middleware에 /admin/* 경로 role="admin" 체크 추가
```

#### 이슈 6: 사용자별 키워드 필터링 미적용
```
현재: dashboard API에서 모든 키워드 반환 (userId 필터 없음)
해결: 로그인 사용자의 userId로 키워드 필터링 적용
```

### 🟢 잘 되어 있는 부분

- **User-Keyword 관계**: Prisma 스키마에 이미 userId 연동 → 본부별 키워드 구현 기반 완비
- **seed.ts**: 10개 본부 계정이 이미 정의되어 있음 → 바로 활용 가능
- **미들웨어 인증**: 세션 쿠키 기반 인증 동작 중 → 확장만 하면 됨
- **API 패턴**: fetchWithRetry, parseItems 등 잘 구조화 → 새 API 추가 시 패턴 재활용
- **UI 컴포넌트**: shadcn/ui 기반 ResultTable, DetailSheet 등 → 재활용 가능

## 4. 에이전트 구성 계획

### 구현에 필요한 에이전트 역할

| 에이전트 | 역할 | 작업 |
|---------|------|------|
| **🏗️ Backend Agent** | API & DB | 낙찰정보API 연동, 스키마 마이그레이션, 알림 코드 제거 |
| **🎨 Frontend Agent** | UI | 관리자 페이지, 분석 대시보드, 사이드바 개편 |
| **🔐 Auth Agent** | 인증/권한 | 미들웨어 role 체크, 관리자 CRUD API, 접속 로그 |
| **📊 Analytics Agent** | 분석 | 수주 분석 API, 차트 컴포넌트, 데이터 집계 |

> 실제로는 Claude가 에이전트팀을 병렬로 구성해서 Phase별로 구현합니다.

- 핵심: **개발 중 `npm run build` 하지 말 것** (HMR 깨짐)
- **TypeScript 우선**, 의존성 변경 시 lockfile 동기화 필수
- 우리 프로젝트에 AGENTS.md 파일 추가하여 에이전트 작업 규칙 명시 예정

  → /admin/analytics 이용현황
  → 기업별/기관별/키워드별 분석 API

Phase 4: 최적화 (30분)
  → 사전규격 용역만 남기기
